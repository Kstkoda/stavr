# github.create_pr

Create a GitHub pull request. Gated by await_decision (CONFIRM tier). On approval, runs `gh pr create`. Returns { ok, pr_url, pr_number } on success.

## Tier behaviour

CONFIRM — opens an `await_decision` first. Only proceeds on approve. On reject/timeout, returns `{ ok: false, reason: ... }` and emits no success event. Trust scopes may grant blanket pre-approval for matching invocations (see [`docs/tool-cards/trust_scope_grant.md`](./trust_scope_grant.md)).

## Side effects

- opens an `await_decision` (CONFIRM gate) unless a matching trust scope authorizes auto-approval
- on approve: shells out to `gh pr create` and emits a `pr_opened` event
- on reject/timeout: no PR created, no `pr_opened` event


## Example

```json
{
  "args": {

    "head": "feat/foo",
    "base": "main",
    "title": "feat: add foo",
    "body": "See spec."
  },
  "result": {

    "correlation_id": "cor_...",
    "pr_url": "https://github.com/Kstkoda/cowire/pull/42",
    "pr_number": 42
  }
}
```

## Error modes

- rejected_by_user (gated decline) → `{ ok: false, reason: "rejected_by_user" }`
- gh failed → `{ code: "gh_failed", ... }`
