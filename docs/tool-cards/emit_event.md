# emit_event

Publish an event. Validates payload against the event taxonomy, persists, fans out to subscribers.

## Tier behaviour

AUTO — runs without confirmation. Read-only or has built-in escape hatches; the caller is responsible for using it intentionally.

## Input schema

```json
{
  "type": "object",
  "properties": {
    "kind": {
      "type": "string"
    },
    "payload": {},
    "correlation_id": {
      "type": "string"
    },
    "tenant_id": {
      "type": "string"
    },
    "source_agent": {
      "type": "string"
    },
    "at": {
      "type": "string",
      "format": "date-time"
    }
  },
  "required": [
    "kind",
    "source_agent"
  ],
  "additionalProperties": false
}
```

## Output schema

```json
{
  "type": "object",
  "properties": {
    "event_id": {
      "type": "string"
    },
    "persisted_at": {
      "type": "string"
    }
  },
  "required": [
    "event_id",
    "persisted_at"
  ],
  "additionalProperties": false
}
```

## Side effects

- writes one row to the events table
- fans out a `notifications/event/published` MCP notification to every subscribed session


## Example

```json
{
  "args": {
    "kind": "progress",
    "payload": {
      "message": "finished phase 2"
    },
    "source_agent": "cc"
  },
  "result": {
    "event_id": "evt_01H...",
    "persisted_at": "2026-05-12T15:00:00.000Z"
  }
}
```

## Error modes

- unknown event kind → tool returns isError with `unknown event kind: <kind>`
- payload does not match the schema for `kind` → `invalid payload for <kind>: ...`
