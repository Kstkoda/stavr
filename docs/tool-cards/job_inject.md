# job_inject

Inject a mid-flight message into a running job. Confirm-tier. The binding must advertise the `inject` capability (see job_list_bindings).

## Tier behaviour

CONFIRM — opens an `await_decision` first. Only proceeds on approve. On reject/timeout, returns `{ ok: false, reason: ... }` and emits no success event. Trust scopes may grant blanket pre-approval for matching invocations (see [`docs/tool-cards/trust_scope_grant.md`](./trust_scope_grant.md)).

## Input schema

```json
{
  "type": "object",
  "properties": {
    "id_or_name": {
      "type": "string",
      "minLength": 1
    },
    "body": {
      "anyOf": [
        {
          "type": "object",
          "additionalProperties": {}
        },
        {
          "type": "string"
        },
        {}
      ]
    }
  },
  "required": [
    "id_or_name"
  ],
  "additionalProperties": false
}
```

## Output schema

```json
{
  "type": "object",
  "properties": {
    "message_id": {

    }
  },
  "required": [

  ],
  "additionalProperties": false

```

## Side effects

- routes the message via the binding handle
- effect on the job is binding-specific


## Error modes

- unknown job → `not_found`
- job not active → `job_inactive`
- binding does not advertise inject → `inject_not_supported`
