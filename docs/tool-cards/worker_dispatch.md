# worker_dispatch

Deliver an instruction to a running worker. Per-spawner tier — some spawners refuse dispatch entirely.

## Tier behaviour

PER-SPAWNER — the tier comes from the spawner registration. Default for built-in spawners (`cc`, `shell`) is CONFIRM.

## Input schema

```json
{
  "type": "object",
  "properties": {
    "id_or_name": {
      "type": "string",
      "minLength": 1
    },
    "body": {
      "anyOf": [
        {
          "type": "object",
          "additionalProperties": {}
        },
        {
          "type": "string"
        },
        {}
      ]
    }
  },
  "required": [
    "id_or_name"
  ],
  "additionalProperties": false
}
```

## Output schema

```json
{
  "type": "object",
  "additionalProperties": {}
}
```

## Side effects

- emits `worker_dispatch_request`
- effect on the worker is spawner-specific


## Error modes

- unknown worker → `unknown_worker`
- spawner does not support dispatch → `dispatch_not_supported`
