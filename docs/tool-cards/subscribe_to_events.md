# subscribe_to_events

Register this MCP session to receive notifications/event/published for the given kinds. Use ["*"] for all.

## Tier behaviour

AUTO — runs without confirmation. Read-only or has built-in escape hatches; the caller is responsible for using it intentionally.

## Input schema

```json
{
  "type": "object",
  "properties": {
    "kinds": {
      "type": "array",
      "items": {
        "type": "string"
      },
      "minItems": 1
    },
    "since_event_id": {
      "type": "string"
    }
  },
  "required": [
    "kinds"
  ],
  "additionalProperties": false
}
```

## Output schema

```json
{
  "type": "object",
  "properties": {
    "subscription_id": {
      "type": "string"
    },
    "kinds": {
      "type": "array",
      "items": {
        "type": "string"
      }
    },
    "replayed_events": {
      "type": "integer",
      "minimum": 0
    }
  },
  "required": [
    "subscription_id",
    "kinds",
    "replayed_events"
  ],
  "additionalProperties": false
}
```

## Side effects

- attaches a subscription to the current MCP session (cleared when the session disconnects)
- if `since_event_id` is set, replays missed events of the requested kinds to the caller


## Example

```json
{
  "args": {
    "kinds": [
      "decision_request",
      "worker_progress"
    ]
  },
  "result": {
    "subscription_id": "sub_...",
    "kinds": [
      "decision_request",
      "worker_progress"
    ],
    "replayed_events": 0
  }
}
```

## Error modes

- empty kinds array rejected by Zod
